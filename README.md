# User Data Dashboard

Symfony dashboard application that visualizes ETL execution results and summary metrics generated by the User Data ETL Service.

## Features

* Displays ETL execution history
* Allows users to select a process execution
* Displays summary metrics for the selected process
* Visualizes data using Chart.js
* Consumes backend REST API endpoints
* Configuration through environment variables
* User registration
* User login
* Session-based authentication
* Dashboard access protection

## Tech Stack

* PHP 8.2
* Symfony 7
* Twig
* Symfony Http Client
* Chart.js
* GitHub
* HTML5

## Architecture

```text
User Data Dashboard
(Symfony + Twig + Chart.js)
          |
          v
User Data ETL Service API
```

## Authentication

The dashboard includes a simple authentication flow:

- User registration (Sign Up)
- User login
- Session-based authentication
- Logout functionality

Only authenticated users can access the dashboard.

## Environment Variables

Configuration is handled through the `.env` file.

```env
BACKEND_BASE_URL=http://127.0.0.1:8000
```

## Installation

Clone the repository:

```bash
git clone https://github.com/edwarsibrian/user-data-dashboard.git
cd user-data-dashboard
```

Install dependencies:

```bash
composer install
```

## Running the Application

Make sure the backend service is running first:

```bash
cd user-data-etl-service
php -S 127.0.0.1:8000 -t public
```

Start the dashboard:

```bash
php -S 127.0.0.1:8001 -t public
```

Open:

```text
http://127.0.0.1:8001/dashboard
```

## Dashboard Features

### Process Selection

The dashboard retrieves available ETL executions from the backend API and allows the user to select a specific process.

### Summary Table

Displays summary metrics returned by the backend service.

### Charts

#### Gender Distribution

Pie chart showing user distribution by gender.

#### Role Distribution

Bar chart showing user distribution by role.

## Backend API Integration

The dashboard consumes the following endpoints:

### List Processes

```http
GET /api/processes
```

### Process Summary

```http
GET /api/processes/{id}/summary
```

## Notes

* The dashboard is intentionally lightweight and focused on data visualization.
* Business logic remains in the backend ETL service.
* Backend URL configuration is externalized through environment variables.
* Charts are rendered using Chart.js loaded from CDN.
* The backend service exposes a dedicated health endpoint for monitoring purposes, but it is not required by the dashboard workflow.
